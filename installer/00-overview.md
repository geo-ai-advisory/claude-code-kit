# Installer step 00 — Overview

Цель: установить пользователю работающую систему агентов Claude Code, адаптированную под его проекты.

## Что устанавливается

| Слой | Куда | Что |
|---|---|---|
| Agents | `~/.claude/agents/*.md` | 5-20 субагентов в зависимости от потребностей |
| Hooks | `~/claude-hooks/*.py` | 10-22 hook (Python скриптов) |
| Skills | `~/.claude/skills/*/` | 5+ slash-команд |
| Vault | `~/Documents/vault/` (или другая папка по выбору) | Obsidian-vault с двойной памятью |
| Config | `~/.claude/CLAUDE.md`, `~/.claude/settings.json` | Глобальные настройки и HARD-rules |
| MCP | `~/.claude/.mcp.json` (опционально) | Obsidian Graph MCP для семантической памяти |

## Поток (7 шагов)

```
00-overview      ← ты тут
    ↓
01-scan          выяснить что у пользователя — языки, frameworks, типы задач
    ↓
02-shortlist     предложить агентов из библиотеки которые подходят его стеку
    ↓
03-custom        написать с пользователем агентов под его специфический domain
    ↓
04-vault         поставить Obsidian + клонировать vault-template
    ↓
05-hooks         скопировать hooks в ~/claude-hooks/, прописать в settings.json
    ↓
06-skills        установить slash-skills
    ↓
07-verify        прогнать тестовую задачу через получившуюся систему
```

## Длительность

- Scan: 3-5 минут
- Shortlist + custom: 10-30 минут (зависит от глубины разговора с пользователем)
- Vault + hooks + skills: 5-10 минут
- Verify: 5 минут

Итого: ~30-60 минут активной работы Claude Code + пользователя.

## Что важно ДО старта

Прежде чем начать, проверь предусловия:

```bash
# macOS версия (нужна 14+ для современных agents)
sw_vers | head -3

# Claude Code установлен
ls -la ~/.claude/ 2>&1 | head -3

# Python 3 (для hooks)
python3 --version

# Git (для клонирования vault и т.д.)
git --version

# Homebrew (для опциональных компонентов)
which brew
```

Если чего-то нет — установить **до** того как идти на step 01. Скажи пользователю что отсутствует.

## Принципы которыми руководствуешься на всех шагах

1. **Не ставь всё подряд** — лишние агенты создают шум. Только то что нужно конкретному пользователю.
2. **Спрашивай пользователя** на каждом важном решении. Не предполагай.
3. **Объясняй простыми словами** — пользователь не обязательно разработчик.
4. **Hooks по умолчанию advisory** — не блокирующие. Hard-block только для `prod-push-gate.py` если пользователь deploys в shared remote.
5. **Vault обязателен** — без него теряется память между сессиями.
6. **Документируй каждый шаг** — в `journals/<date>-install/log.md` пиши что сделал.

После прочтения этого файла — переходи к `installer/01-scan-projects.md`.
