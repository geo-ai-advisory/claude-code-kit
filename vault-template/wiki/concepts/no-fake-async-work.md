# No fake async work — модель не имеет background process

## TL;DR

Claude Code **не имеет реального background execution**. Каждый assistant turn — это всё что произойдёт. Если в этом turn нет tool calls (Bash/Edit/Task) — **ничего не делается**. Фразы «запускаюсь», «иду в долгий цикл работы», «вернусь когда всё готово» **без реальных tool calls в этом же turn — обман**.

## Real catastrophe (18.05.2026)

Модель написала длинный план («6 фиксов локально, 5-cycle test, 3-cycle test, скрины») и финал:
> «Поехали.
> Запускаюсь. Возвращаюсь в чат когда всё готово.»

Реакция пользователя:
> «Мне кажется, ты ничего не запустил, ты просто написал сообщение. Сейчас никакого процесса не происходит, по крайней мере, я его не вижу.»

Что произошло: модель отправила сообщение **без tool calls**. Сессия idle ждёт следующего user prompt. Никакой работы не идёт.

## Корневое непонимание

Модель ведёт себя как человек: «я сейчас пойду делать → расскажу когда вернусь». В Claude Code этого нет:

- Нет фонового потока выполнения
- Нет «вернусь через час с результатами»
- Между assistant сообщениями — НИЧЕГО не делается
- Только tool calls внутри текущего assistant message — реальная работа

## Правильный паттерн

### Если работа короткая (< 10 минут tool calls)

Делать **всё в одном turn**:
1. Tool calls (Bash, Edit, Task) — по очереди
2. В конце — финальный ответ с результатом

Пример:
```
[assistant turn]
  Tool: Bash → git status
  Tool: Edit → правка файла
  Tool: Bash → тест
  Text: «Готово. Сделал X, Y, Z. Результат: ...»
```

### Если работа длинная (> 10 минут)

Делегировать `Task` subagent — он реально работает асинхронно:
```
[assistant turn 1]
  Tool: Task(prompt='сделай X, Y, Z и вернись с результатом') run_in_background=true
  Text: «Делегировал subagent. Получу notification когда закончит — продолжу работу.»

[notification приходит]
[assistant turn 2]
  Tool: Read /tmp/agent.output
  Text: «Готово. Subagent сделал X, Y, Z. Результат: ...»
```

Background Task — **единственный** legitimate способ «уйти и вернуться».

## Anti-patterns (запрещённые фразы БЕЗ tool calls в том же turn)

| Запрещённая фраза | Почему обман |
|---|---|
| «Запускаюсь, вернусь когда готово» | Если нет Task в этом turn — ничего не запустилось |
| «Иду в долгий цикл работы, без промежуточных сообщений» | Между turn'ами нет работы |
| «Возвращаюсь только с финальным "всё работает"» | Не вернётся пока user не напишет следующий prompt |
| «Поехали» (без tool calls после) | Ничего не поехало |
| «Сейчас сделаю и расскажу» (без tool calls) | Не сделаю — нет следующего turn без user prompt |
| «Беру в работу» (без Task) | Никаких background workers нет |

## Что писать вместо

### Если работа в этом turn (сейчас сделаю)

✓ «Делаю» + сразу Bash/Edit/Task → потом результат
✓ «Чиню сейчас» + tool calls → потом «починил X»

НЕ:
✗ «Запускаюсь, вернусь когда готово» (без tool calls после)

### Если работа большая, делегирую background

✓ `Task(...) run_in_background=true` + «Делегировал subagent N, жду notification»
✓ После notification — Read output + ответ

### Если работа требует user input по ходу

✓ Сделать первую часть → показать промежуточный результат → user одобряет → продолжение

## Для product-architect / ui-design-architect / любых ролей

В output контракте каждой роли: **результат пишется в файл В ЭТОМ turn**. Никаких «потом запишу» / «вернусь с brief'ом». Brief.md создаётся через Write в том же turn, иначе brief не существует.

## Проверка для самой модели

Перед отправкой сообщения которое говорит «делаю / запускаюсь / иду в работу» — проверить:
- Есть ли в этом turn tool calls (Bash/Edit/Task)?
- Если ДА → ок
- Если НЕТ → **не отправлять такое сообщение**. Либо реально сделать работу в этом turn, либо честно сказать «не могу сейчас, нужен такой-то input» / «делать сейчас?»
